Software Testing Lifecycle (STLC)

It is a set of process executed in a sequence, systematic and planned manner in-order to achieve the quality and testing goal. Each activity is carried out in a Planned and sytematic manner way with each phase have a different goals and results

Stages of Software Testing Life Cycle

  • Requirement Analysis
  • Test Planning
  • Test Design
  • Test Environment Setup
  • Test Execution
  • Bug Reporting and Management
  • Requirement traceability matrix
  • Test Closure

    • Requiremnt Analysis

    Analyze the business requirement and plan all the testing activities to be done to deliver a quality Test strategy,analyze the risks (tools,resource and time )Schedule and time line deliverable

    Test Planning

    Test planning is the first step in testing process which is based on the Requirement analysis The test plan varry according to the company and product but there is an IEEE Standard for test plan

  • Test plan Identifier - A unique plane name
  • Introduction - Overview and objectives of test plan
  • References - Project plan related documents
  • Test items - Product or Software with detailed Specification
  • Features to be Tested
  • Features not to be Tested
  • Approach
  • Testcase status - Pass,Fail,skip..
  • Test Deliverables

    • Test Design

    The test case design should cover all the aspects Specifications such as boundary value analysis,equivalence partioning, decision table,state transition and usecase testing Structure based such as statement testing(covers each lines of code like unit testing),Decission testing( It covers all the branch of codes),Condition testing Experience based such as guessing the error and explotary testing

    Test Environment Setup

    The Environment setups such as System,database,network,Operating system, and Other Components with licsnse according to the requirement of the project

    Test execution

    The test execution is the process of execution of the Program or software or and check wether the Expected and Actual results are same

    Bug Reporting and Management

    During the testing process many bugs may come then the tester will take the screenshot or Video logs of the Application or software under Test and Detailed specification of the environment setup and riase an issue in Bug Reporting tool. Once the developer solves the issue then the tester retest again the application or software under test and update the test status

    Requirement traceability matrix

    Requirement traceability matrix is mostly in the form of table which contains the details of each phase of testing met the requirement of the project

    Test Closure

    The Test Closure is a phase which contains no of cases tested
